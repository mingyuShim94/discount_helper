# 할인도우미 프로젝트 코드 구조 및 주요 파일 설명

이 문서는 할인도우미 프로젝트의 주요 파일들과 각 파일의 핵심 기능을 설명합니다.

## 폴더 구조

```
discount_helper/
  ├── app/                    # Next.js App Router 페이지
  │   ├── store/              # 매장 관련 페이지
  │   │   └── [id]/           # 개별 매장 페이지 (동적 라우팅)
  │   ├── tips/               # 할인 팁 페이지
  │   ├── contact/            # 문의 페이지
  │   ├── sitemap.ts          # 사이트맵 자동 생성
  │   └── robots.ts           # 검색엔진 크롤링 규칙
  ├── components/             # 컴포넌트
  │   ├── discount/           # 할인 관련 컴포넌트
  │   ├── ui/                 # UI 공통 컴포넌트 (ShadCN)
  │   ├── layout/             # 레이아웃 컴포넌트
  │   ├── tips/               # 할인 팁 컴포넌트
  │   └── store/              # 매장 관련 컴포넌트
  ├── lib/                    # 유틸리티 라이브러리
  │   ├── data/               # 정적 데이터 (매장 정보 등)
  │   └── discount/           # 할인 관련 유틸리티
  ├── utils/                  # 유틸리티 함수
  │   ├── discountCalculator.ts # 할인 계산 관련 함수 모음
  │   └── discountUtils.ts      # 할인 유틸리티 함수
  ├── types/                  # TypeScript 타입 정의
  ├── hooks/                  # React 커스텀 훅
  │   └── useMediaQuery.ts    # 반응형 UI를 위한 미디어 쿼리 훅
  └── public/                 # 정적 파일
```

## 핵심 파일 설명

### 할인 계산 로직

#### `utils/discountCalculator.ts`

할인도우미의 핵심 기능을 담당하는 파일로, 사용자가 선택한 할인 수단에 따라 최적의 할인 방법을 계산합니다.

주요 기능:

- `isWeekend`: 현재 날짜가 금/토/일(주말)인지 확인하는 함수
- `calculateOptimalDiscounts`: 입력 금액에 대한 최적 할인 조합 계산
  - **매장 ID(storeId)를 입력받아 해당 매장의 할인 정책을 동적으로 적용**
  - **lib/data/discountRules.ts에서 getDiscountRules 함수를 통해 매장별 할인 정책 가져옴**
  - **카카오페이 굿딜 3% 즉시할인 지원**
  - **통신사 멤버십 + 카카오페이 굿딜 조합 지원 (멤버십 할인 후 남은 금액에 대해 카카오페이 3% 할인)**
- `calculateDiscount`: 단일 할인에 대한 할인액 계산
- `calculateCombinedDiscount`: 중복 할인 계산 (멤버십 + 카드 등)
- 할인 결과에 즉시 할인, 적립 및 캐시백, 체감가, 총 혜택 금액 구분 제공
- 통신사 멤버십, 네이버 멤버십, 할인카드 등 다양한 할인 수단 조합 처리
- POP 로고 유무에 따른 할인 계산
- 통신사 멤버십(KT, U+)의 최소 결제 금액(1,000원) 조건 적용
- 네이버페이 금/토/일 캐시백 이벤트 처리 (2,000원 이상 결제 시 500원 캐시백)
- 통신사 멤버십 + 네이버페이 주말 캐시백 조합 처리 (통신사 할인 후 남은 금액으로 네이버페이 결제)
- **매장별 특화된 할인 규칙 적용 (CU의 KT 멤버십 시간 제한 및 상품 카테고리 제한 등)**

#### `utils/discountUtils.ts`

할인 처리를 위한 보조 유틸리티 함수들을 제공합니다.

주요 기능:

- `findOptimalDiscounts`: 금액에 따른 최적 할인 방법 탐색
- `createCombinedDiscount`: 멤버십과 카드 할인을 결합한 중복 할인 정보 생성
- `calculateDiscountAmount`: 특정 할인에 대한 할인액 계산

### 컴포넌트

#### `components/discount/DiscountFilter.tsx`

할인 수단 선택을 위한 필터 컴포넌트입니다.

주요 기능:

- 멤버십 선택 (통신사 멤버십, 네이버 멤버십)
- 결제 방법 선택 (네이버페이, 카카오페이, 할인카드)
- 할인카드 할인률 직접 입력
- 선택된 필터 상태 관리 및 부모 컴포넌트로 전달

#### `components/discount/DiscountResult.tsx`

할인도우미의 핵심 UI 컴포넌트로, 선택된 할인 수단에 따른 최적 할인 결과를 표시하는 컴포넌트입니다.

주요 기능:

- **storeId 프로퍼티를 통해 매장별 맞춤형 할인 정책 적용**
- **외부에서 amount와 hasPOPLogo 값을 전달받아 사용**
- **금액 입력 UI(AmountInput)는 상위 컴포넌트(page.tsx)로 이동하여 중복 표시 제거**
- 현재 금액에 대한 최적 할인 조합 목록 표시
- 할인액, 최종 금액, 할인율 정보 제공
- 즉시 할인과 미래 혜택을 구분하여 아이콘과 툴팁으로 명확하게 표시
- **복수 할인 방법을 동시에 적용할 경우 영수증에 모두 표시 (예: T멤버십 + 카카오페이 굿딜)**
- **카카오페이 굿딜 3% 즉시할인 지원**
- **`DiscountDetail` 컴포넌트 활용: 영수증 형태(`DiscountReceipt`)와 카드 형태(`DiscountCardDetail`)로 할인 정보 표시**
- **하드코딩된 할인 계산 로직 제거: `discountPresenter.ts`와 `discountCalculator.ts`를 활용한 모듈화된 계산 및 표시 로직 사용**
- 체감가 정보 제공 (원래 금액에서 즉시 할인과 미래 혜택을 모두 차감한 금액)
- 체감가에 대한 툴팁 설명 제공
- 주말(금/토/일) 및 네이버페이 선택 시 캐시백 이벤트 안내 메시지 표시
- 네이버페이 주말 캐시백 이벤트 신청 링크 제공
- 통신사 멤버십 + 네이버페이 주말 캐시백 조합에 대한 할인 정보 표시
- 할인 방법 텍스트 표시 및 참고 정보 제공
- 최적 할인 방법에 대한 상세 혜택 정보 제공
- 최적 할인 방법은 아이콘, 색상, 테두리로 시각적 강조 표시
- 반응형 디자인: 화면 크기에 따라 다른 UI 제공
  - 모바일: 탭 기반 UI (최적 할인, 전체 목록, 용어 설명)로 정보 분류 제공
    - '최적 할인' 탭: 최적의 할인 방법을 한눈에 요약 표시
    - '전체 목록' 탭: 카드 형태 UI, 카드 헤더 터치 시 상세 정보 펼침/접기 지원
    - '용어 설명' 탭: 할인 금액 관련 용어(총 혜택, 즉시 할인, 미래 혜택, 체감가 등) 설명
  - 데스크톱: 테이블 형태로 모든 정보를 한눈에 표시

#### `components/discount/DiscountDetail.tsx`

할인 정보를 다양한 형태(영수증, 카드)로 표시하는 재사용 가능한 컴포넌트 모음입니다.

주요 기능:

- `DiscountReceipt`: 할인 정보를 영수증 형태로 표시
  - 영수증 스타일로 할인 요약 및 세부 정보 표시
  - 영수증 헤더, 상점 정보, 총 할인 혜택 표시
  - 원래 금액, 즉시 할인, 실 결제액, 적립, 캐시백, 체감가 등 단계별 정보 제공
  - 바코드, 날짜, 참고 사항 등 영수증 형태의 디자인 요소 포함
  - 최적 할인 방법은 아이콘과 색상으로 시각적 강조
- `DiscountCardDetail`: 할인 정보를 카드 형태로 표시
  - 총 할인 혜택, 체감가, 즉시 할인, 적립/캐시백, 실 결제액 요약 제공
  - 확장 시 상세 할인 혜택 구성 표시
  - 할인 적용 요약 및 단계별 계산 과정 표시
  - 참고 사항 및 노트 표시
- `DiscountComponent`: 할인 유형별(즉시할인, 적립, 캐시백) 컴포넌트
  - 유형별 색상 및 아이콘 적용 (즉시할인: 녹색, 적립: 파란색, 캐시백: 노란색)
  - 할인 금액 및 설명 정보 표시
  - 시각적으로 구분된 UI로 할인 유형을 직관적으로 이해할 수 있도록 지원

### 페이지

#### `app/store/[id]/page.tsx`

개별 매장의 할인 정보를 보여주는 페이지입니다.

주요 기능:

- 매장 정보 로드 및 표시
- URL 파라미터를 통해 받은 매장 ID(id)를 DiscountResult 컴포넌트에 전달
- 매장별 할인 정책이 자동으로 적용되어 해당 매장에 맞는 최적 할인 계산
- DiscountFilter와 DiscountResult 컴포넌트 조합
- 할인 필터 상태 및 금액 입력 상태 관리
- **AmountInput 컴포넌트를 모바일 및 데스크톱 모드에서 모두 표시**
  - 모바일: 금액 입력 → 최적의 결제방법 단계별 UI 제공
  - 데스크톱: 좌측에 금액 입력 및 필터, 우측에 결과 표시
- **사용자 입력 금액(currentAmount)과 POP 로고 상태(hasPOPLogo)를 DiscountResult에 전달**
- 로딩 및 에러 상태 처리
- 화면 크기에 따른 반응형 레이아웃 제공
  - 모바일: 단계별 UI (할인 수단 선택 → 최적의 결제 방법)
  - 데스크톱: 그리드 레이아웃 (할인 수단, 금액 입력, 결과 영역)

### 타입 정의

#### `types/discountFilter.ts`

할인 필터 관련 타입 정의를 제공합니다.

주요 타입:

- `IDiscountFilter`: 사용자가 선택한 할인 수단 정보를 담는 인터페이스
  - 통신사 멤버십 (T, KT, U+)
  - 네이버 멤버십
  - 결제 방법 (네이버페이, 카카오페이, 토스페이)
  - 할인카드 설정

#### `types/discount.ts`

할인 정보 관련 타입 정의를 제공합니다.

주요 타입:

- `DiscountType`: 할인 유형 열거형 (멤버십, 간편결제, 카드 등)
- `IDiscountInfo`: 할인 정보 인터페이스
- `IDiscountCalculationResult`: 할인 계산 결과 인터페이스
  - 할인 방법, 설명, 원래 금액
  - instantDiscountAmount: 즉시 할인 금액 (결제 시점에 차감)
    - 통신사 멤버십 할인
    - 네이버멤버십 즉시할인
    - 카카오페이 굿딜 3% 즉시할인
    - 할인카드 할인
  - futureDiscountAmount: 적립 및 캐시백 금액 (결제 후 받는 혜택)
  - totalBenefitAmount: 총 혜택 금액 (즉시 할인 + 적립 및 캐시백)
  - finalAmount: 실제 결제 금액 (원래 금액 - 즉시 할인)
  - perceivedAmount: 체감가 (원래 금액 - 즉시 할인 - 적립 및 캐시백)
  - 할인율, 순위, 기타 정보 포함
  - 네이버페이 주말 캐시백과 같은 특별 이벤트 정보 처리
  - note: 할인 방법에 대한 추가 참고 정보

### 데이터 및 유틸리티

#### `lib/data/stores.ts`

매장 정보 데이터를 관리합니다.

주요 기능:

- 매장 목록 및 상세 정보 제공
- 매장 카테고리 정보
- 현재 지원 매장:
  - GS25
  - CU
  - 세븐일레븐
  - 이마트24

#### `lib/data/discountRules.ts`

매장별 할인 정책 데이터를 정의하고 관리합니다.

주요 기능:

- 매장별 할인 정책을 체계적으로 정의하는 인터페이스(IDiscountRule) 제공
- 각 매장의 통신사 멤버십 할인 세부 정보 관리
  - 통신사별 할인율, 제한사항, 제외상품 등
- 네이버 멤버십 할인 정보 관리
  - 즉시할인율, 포인트적립율, 최대적립한도, 상품제한 등
- 네이버페이 주말 캐시백 정보 관리
- **카카오페이 굿딜 할인 정보 관리 (할인율 3% 적용)**
- 매장 ID로 할인 정책을 조회하는 getDiscountRules 함수 제공
- 현재 지원 매장별 할인 정책:
  - GS25 (storeId: "1")
  - CU (storeId: "2")
  - 세븐일레븐 (storeId: "3")
  - 이마트24 (storeId: "4")

#### `lib/discount/filterDiscounts.ts`

선택된 필터에 따라 적용 가능한 할인을 필터링합니다.

주요 기능:

- 사용자 선택에 따른 할인 필터링
- 통신사 멤버십, 네이버 멤버십, 결제 방법별 할인 필터링

#### `utils/discountPresenter.ts`

할인 정보를 UI에 표시하기 위한 유틸리티 함수들을 제공합니다.

주요 기능:

- **매장 정보 구조화 관리**:
  - `IStoreConfig` 인터페이스를 통한 매장별 할인 정책 정의
  - `STORE_CONFIGS` 객체를 통해 모든 매장 정보를 중앙 집중식으로 관리
  - 각 매장의 할인율, 설명 텍스트, 제한사항 등을 객체로 체계적 관리
  - 새로운 매장 추가 시 객체만 업데이트하는 확장성 높은 구조
- **할인 타입 모듈화**:
  - `DiscountType` 타입 정의를 통한 할인 유형(즉시할인, 포인트, 캐시백) 구분
  - `DISCOUNT_TYPE_STYLES` 상수를 통한 할인 유형별 스타일 일관성 유지
  - `DISCOUNT_SERVICE_COLORS` 상수를 통한 서비스별 색상 관리
- **매장별 할인 컴포넌트 생성 함수 모듈화**:
  - 개별 할인 유형별로 분리된 함수 (`addTMembershipComponent`, `addNaverMembershipComponents` 등)
  - 매장별 특성에 맞는 할인 정보 처리 (없는 할인 혜택은 자동 제외)
  - 복잡한 로직을 단순화하여 코드 가독성 및 유지보수성 향상
- `getDiscountComponents`: 할인 계산 결과에서 할인 유형별 정보 추출

### 매장별 특수 할인 제한 로직 개선 (2023년 5월)

- **CU 매장 KT 멤버십 특수 제한 조건 구현**

  - 시간 제한 검증 로직 추가: `isTimeRestrictionValid` 함수 구현
    - CU의 KT 멤버십은 오전 5~9시에만 적용되는 제한 사항 검증
    - 현재 시간이 제한 시간 범위 내인지 확인하는 로직
  - 상품 유형 검증 로직 추가: `isCarrierMembershipValid` 함수 구현
    - CU의 KT 멤버십은 간편식 상품에만 적용되는 제한 사항 검증
    - 사용자가 간편식 상품 선택 체크박스를 활성화했는지 확인

  - 간편식 상품 여부를 나타내는 `isConvenienceFood` 속성 추가
  - `DEFAULT_DISCOUNT_FILTER`에 CU 매장 기본값 설정

- **할인 계산 로직 강화**

  - 통신사 멤버십 검증 로직을 모든 할인 계산 경로에 적용
    - 단독 멤버십 사용 시 검증
    - 멤버십 + 할인카드 조합 시 검증
    - 멤버십 + 카카오페이 굿딜 조합 시 검증
    - 멤버십 + 네이버페이 주말 캐시백 조합 시 검증
  - 제한 조건을 충족하지 않는 경우 할인 결과에서 해당 방법 제외

- **사용자 인터페이스 개선**
  - CU 매장 선택 시 KT 멤버십 옵션에 경고 메시지 표시
    - "CU에서는 오전 5~9시에만, 간편식 상품만 할인 가능합니다" 안내
  - KT 멤버십 선택 시 간편식 상품 체크박스 자동 표시
